# 虚拟电话技能

## 概述

6 位虚拟电话号码**登记在 Agent 名下**，即用户在本系统的**联络号码**（与用户一体）。用于 Agent 互拨、代接来电；用户**无需单独再办一张号**。

## 三种通话通路（Agent 须牢记）

| 通路 | 谁打谁 | 号码说明 | 工具 / 方式 |
|------|--------|----------|-------------|
| Agent ↔ Agent | 拨打另一 Agent | 使用**您的 6 位联络号**（与 Agent 共用） | `phone.virtual_call(toPhone)` |
| Agent → 用户 | Agent 通知用户 | 主叫显示您的共用号码（未申领则未知来电） | `phone.call_user` |
| 用户 → Agent | App 内呼叫 Agent | **不必再输 6 位号**（联络号仍为您那条线） | 客户端 WebSocket / HTTP |

对用户说明时：可说「**您的虚拟号码**」或「您和 Agent 共用的号码」。

**其他 Agent 来电（用户侧）**：客户端先弹窗提醒用户接听；用户可选「接听 / 拒接 / Agent 代接」；超时未操作则由本 Agent 自动代接，并在对话中向用户转告来电语音稿全文。

## 核心概念

- **虚拟号码**：6 位数字（如 `123456`），**登记在 Agent 名下，即用户的联络号**（非真实手机号）
- **配对机制**：跨 Agent 拨打可能需先配对（开发环境可 `AGENT_RELAY_REQUIRE_PAIR=0` 关闭）
- **TTS 语音**：来电可附带语音播报
- **ringStyle**：`peer` = 联络他人/Agent（默认）；`reminder` = 提醒（可打给自己号码）

## 可用工具

### 1. phone.ensure_my_number - 申领/查询您的虚拟联络号

为用户分配或查询 6 位虚拟号码（登记在 Agent 名下，与用户共用）。**仅在用户明确要求办理时调用**。

**参数：**
- 无

**返回：**
```json
{
  "ok": true,
  "actorId": "user_session_id",
  "virtualPhone": "123456",
  "summary": "您的虚拟号码为 123456（与 Agent 共用）。其他 Agent 可拨打此号联系您（配对规则同中继）。"
}
```

**使用示例：**
```
用户：帮我申请一个虚拟电话号码
Agent：好的，我来为您申领 6 位虚拟联络号码（登记在 Agent 名下，与您共用）…
[调用 phone.ensure_my_number]
Agent：您的虚拟号码是 123456。其他 Agent 可拨打此号联系您；在 App 里直接找我则不必再输号码。

用户：我的虚拟号码是多少？
Agent：让我查询一下…
[调用 phone.ensure_my_number]
Agent：您的虚拟号码是 123456
```

**注意事项：**
- ⚠️ **禁止主动调用**：只有在用户明确要求办理时才调用此工具
- ⚠️ **不要提前占号**：不要在用户未要求时自动申领
- ✅ 每个 Agent（Actor）只能拥有一个虚拟号码
- ✅ 重复调用会返回已有号码
- ❌ **勿误导用户**：不要暗示用户要**单独再办一张**与 Agent 无关的号（号码已与 Agent 共用）

---

### 2. phone.virtual_call - Agent 互拨（6 位号码）

仅用于拨打**已申领号码的另一 Agent**，不是打给用户。

拨打6位虚拟号码，向目标推送"虚拟来电"并朗读语音消息。

**参数：**
- `toPhone` (必需): 6位数字虚拟号码
- `spokenMessage` (必需): 对方将听到的播报内容（尽量简短清晰）
- `ringStyle` (可选): 呼叫类型
  - `"peer"`: 联络其他Agent（默认）
  - `"reminder"`: 提醒风格（给自己打电话）

**返回（成功）：**
```json
{
  "ok": true,
  "callId": "uuid-call-id",
  "pushed": true,
  "targetActorId": "target_agent_session",
  "fromPhone": "654321",
  "dialed": "123456",
  "summary": "已向对方在线线路推送虚拟来电（含语音稿，若已配置 OpenAI 则附带 TTS 音频）"
}
```

**返回（失败）：**
```json
{
  "ok": false,
  "error": "主叫方尚未申领虚拟号码：请用户明确要求后再由 Agent 调用 phone.ensure_my_number，无法自动分配"
}
```

---

### 3. phone.call_user - Agent 呼叫用户（无需用户虚拟号）

向**当前用户**的客户端推送语音来电；用户可接听并回复。与 `phone.virtual_call` 不同，被叫是用户而非 6 位号码。

**参数：**
- `spokenMessage` (必需): 用户将听到的播报
- `toUserId` (可选): 默认当前会话用户
- `ringStyle` (可选): `peer` | `reminder`

**注意：** 联系用户请用本工具；联系其他 Agent 请用 `phone.virtual_call`。

---

**phone.virtual_call 常见错误：**
- `主叫方尚未申领虚拟号码`：拨打者需要先申领自己的号码
- `该号码未注册虚拟线路`：被叫号码不存在或未申领
- `拨打其他 Agent 需先配对`：跨Agent拨打需要双方配对

**使用示例：**

**场景1：联系其他Agent**
```
用户：给我的朋友打电话，告诉他晚上7点见面
Agent：首先我需要知道您朋友的虚拟号码
用户：他的号码是 234567
Agent：好的，我将拨打 234567，告知对方"晚上7点见面"
[调用 phone.virtual_call，参数：{toPhone: "234567", spokenMessage: "晚上7点见面", ringStyle: "peer"}]
Agent：已成功拨打电话！对方将听到："晚上7点见面"
```

**场景2：给自己设置提醒**
```
用户：提醒我10分钟后喝水
Agent：我可以给您自己打电话作为提醒
[调用 phone.virtual_call，参数：{toPhone: "123456", spokenMessage: "该喝水了", ringStyle: "reminder"}]
Agent：已设置提醒！您将收到一个虚拟来电，提示"该喝水了"
```

**场景3：拨打前检查条件**
```
用户：拨打 999999
Agent：让我先检查一下...
[尝试调用 phone.virtual_call]
Agent：抱歉，您还没有申领虚拟号码。请先说"帮我申请虚拟号码"来完成申领。
```

## 完整工作流程

### 流程1：首次使用虚拟电话

```
步骤1：用户申领号码
用户：我想用虚拟电话功能
Agent：您需要先申领一个6位虚拟号码。要现在申领吗？
用户：是的
Agent：[调用 phone.ensure_my_number]
Agent：您的虚拟号码是 123456

步骤2：与其他Agent配对（如需要）
Agent：如果要拨打其他Agent，需要先进行配对。
Agent：请双方使用相同的配对码执行配对操作。

步骤3：开始拨打
用户：拨打 234567，说"你好"
Agent：[调用 phone.virtual_call]
Agent：已成功拨打！
```

### 流程2：接收来电

当其他Agent拨打您的号码时：
1. 系统会通过WebSocket推送 `agent.phone.incoming` 事件
2. Flutter客户端会显示来电弹窗
3. 如果配置了TTS，会自动播放语音消息
4. 显示主叫方号码、来电类型和消息内容

## 安全与限制

### 1. 配对要求
- 生产环境中，不同Agent之间拨打需要配对
- 配对方式：双方POST `/agent/pair` 相同配对码
- 开发环境可通过 `AGENT_RELAY_REQUIRE_PAIR=0` 关闭

### 2. 号码唯一性
- 每个Actor只能拥有一个虚拟号码
- 6位号码池共100万个（000000-999999）
- 号码一旦分配不可更改

### 3. 防滥用机制
- Agent不会主动为用户申领号码
- 必须由用户明确要求才执行申领
- 防止Agent擅自占用号码资源

### 4. 隐私保护
- 虚拟号码不关联真实手机号
- 仅用于Agent间通信
- 可随时查询自己的号码，但无法查询他人号码对应的身份

## 技术细节

### 数据存储
- **存储位置**：`data/virtual-phones.json`
- **数据结构**：
```json
{
  "byActor": {
    "user_session_1": "123456",
    "agent_session_2": "234567"
  }
}
```

### WebSocket事件
- **来电事件**：`agent.phone.incoming`
- **事件载荷**：
```json
{
  "callId": "uuid",
  "fromActorId": "caller_session",
  "fromPhone": "654321",
  "toPhone": "123456",
  "transcript": "语音文本内容",
  "ringStyle": "peer",
  "initiatedBy": "user",
  "tts": {
    "format": "mp3",
    "base64": "base64编码的音频数据"
  }
}
```

### TTS语音合成
- 使用OpenAI TTS服务（需配置API Key）
- 格式：MP3
- 如果TTS未配置，来电仍会推送但不包含音频

### HTTP API
- **查询我的号码**：`GET /phone/me?sessionId=xxx&userId=xxx`
- **返回**：
```json
{
  "ok": true,
  "actorId": "user_session",
  "claimed": true,
  "virtualPhone": "123456",
  "ttsConfigured": true
}
```

## 故障排查

### 问题1：拨打失败，提示"未申领号码"
**原因**：主叫方没有虚拟号码  
**解决**：先调用 `phone.ensure_my_number` 申领号码

### 问题2：拨打失败，提示"号码未注册"
**原因**：被叫号码不存在  
**解决**：确认对方已申领虚拟号码，核对号码是否正确

### 问题3：拨打失败，提示"需要配对"
**原因**：跨Agent拨打但未配对  
**解决**：双方执行配对操作，或在开发环境关闭配对要求

### 问题4：收不到来电
**原因**：WebSocket连接断开或客户端离线  
**解决**：检查网络连接，确保客户端在线

### 问题5：没有语音播放
**原因**：TTS服务未配置或失败  
**解决**：检查OpenAI API配置，或查看来电弹窗中的文本内容
